# 100 Tool Backlog

## Purpose

This backlog defines a long-term target catalog for the toolbox. It is designed for a future scale of 100+ tools, not only the current 15-tool state.

The list is based on common patterns from CTool, DevToys, CyberChef, IT-Tools, DevUtils, FreeFormatter, utils.live, and similar developer tool collections. It favors local-first browser utilities: tools that can usually run without sending user input to a server.

## Source Notes

- CTool emphasizes everyday development utilities such as hash/encryption, encoding conversion, timestamps, QR code, pinyin, IP lookup, code optimization, Unicode, regex, PHP serialization, and text diff.
- DevToys groups tools around converters, encoders/decoders, formatters, generators, testers, text utilities, graphics, JSON/YAML, date, number bases, JWT, QR code, hash, lorem ipsum, password, markdown, regex, SQL, XML, and text comparison.
- CyberChef demonstrates the high-scale model: hundreds of categorized operations, searchable, composable, and focused on encoding, decoding, crypto, extraction, compression, and transformation.
- IT-Tools, DevUtils, FreeFormatter, utils.live, and similar web toolboxes confirm the same core demand: JSON, XML, SQL, encoders, JWT, hash, cron, UUID, color, text, URL, API, database, and file/data conversion tools.

## Category Model

The current app categories can support the first 100 tools if `group` remains the second-level taxonomy:

- `format`: data/code formatting and validation.
- `encode`: encoding, escaping, compression, and representation conversion.
- `time`: dates, schedules, durations, and clocks.
- `security`: crypto, tokens, keys, certificates, and security inspection.
- `generate`: IDs, passwords, mock data, test data, and scaffolding.
- `code`: API/code/schema helpers and developer workflow converters.
- `text`: text analysis, comparison, extraction, and list utilities.

If the catalog grows beyond 150-200 tools, consider splitting `network`, `database`, and `media` into first-class categories. For now they can live under `code`, `format`, `security`, or `text` with clear `group` values.

## Implementation Priority

- `P0`: Core daily developer workflow. Build first.
- `P1`: Strongly useful, slightly more complex or narrower audience.
- `P2`: Valuable but can wait, often needs larger libraries or data tables.

## Notes for Future Registry Entries

Each new tool should include:

```ts
{
  id: 'stable-kebab-id',
  title: '中文工具名',
  description: '一句话说明输入、输出和本地处理。',
  path: '/tools/stable-kebab-id',
  category: 'format',
  group: 'Specific Group',
  keywords: ['primary', 'search', 'terms'],
  aliases: ['common alternate names'],
  featured: false,
  order: 10,
  status: 'active',
  component: () => import('./stable-kebab-id/StableTool.vue'),
}
```

Avoid adding tools without `group`, `keywords`, `aliases`, and `order`; the directory depends on those fields to remain scalable.
